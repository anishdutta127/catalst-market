# How to regenerate `design-system/MASTER.md`

`MASTER.md` is the source of truth for Catalst Market's design system. It was generated by the **ui-ux-pro-max** skill (`uipro` CLI) and then **hand-tuned** to honor the hard constraints in `CLAUDE.md` that the skill's database doesn't know about.

If you ever need to regenerate it from scratch — or run a fresh search to compare alternatives — follow this recipe. **Do not blindly overwrite `MASTER.md` with the raw output.** The skill picks landing-page patterns and Roboto by default; both are wrong here.

---

## Prerequisites

- **Node.js + npm** (for the `uipro` CLI installer)
- **Python 3.10+** (the skill's search scripts are Python)

Verify:

```bash
node --version    # >= 18
npm --version
python --version  # 3.10+ (we tested on 3.12)
```

If Python isn't on PATH on Windows, install via `winget install Python.Python.3.12` and reopen the shell.

---

## 1. Install the skill

Install the `uipro` CLI globally, then bootstrap the skill into this project:

```bash
npm install -g uipro
uipro init -a claude
```

This creates `.claude/skills/ui-ux-pro-max/` (committed to the repo) containing the search scripts and the curated CSV databases (67 styles, 96 palettes, 57 font pairings, 25 chart types, etc.). It also writes `.claude/settings.local.json` for local Claude Code permissions — that file is gitignored.

If `.claude/skills/ui-ux-pro-max/` is already present (it is, after the initial commit), you can skip this step. Re-run `uipro init -a claude --force` only when you want to upgrade the bundled assets.

---

## 2. Run the design-system search

This is the exact command that produced the current `MASTER.md`:

```bash
python .claude/skills/ui-ux-pro-max/scripts/search.py \
  "editorial consumer feed mobile lifestyle storytelling startup news mood pinterest substack" \
  --design-system \
  --persist \
  -p "Catalst Market" \
  -f markdown
```

What this does:

- Searches the `product`, `style`, `color`, `landing`, and `typography` databases in parallel.
- Picks the highest-ranked match in each domain and assembles a recommended pattern + palette + typography + effects + anti-patterns.
- `--persist` writes the result to `design-system/catalst-market/MASTER.md` (a sub-directory — note this differs from our canonical path).
- `-f markdown` formats for documentation rather than terminal display.

To explore alternatives without writing anything, drop `--persist` and re-run with different keywords:

```bash
# Try a different angle
python .claude/skills/ui-ux-pro-max/scripts/search.py \
  "consumer news feed mobile social editorial magazine pinterest masonry cards" \
  --design-system -f markdown

# Domain-specific deep dives
python .claude/skills/ui-ux-pro-max/scripts/search.py "editorial magazine serif modern" --domain typography -n 8
python .claude/skills/ui-ux-pro-max/scripts/search.py "consumer feed gen-z" --domain color -n 6
```

## 3. Hand-tune against `CLAUDE.md` hard constraints

**This step is mandatory.** The raw skill output will violate these every time:

| Skill default | Why it's wrong here | Replace with |
|---|---|---|
| Body font: **Roboto** | Banned in `CLAUDE.md` ("refuse on sight: Inter or Roboto") | **Geist Sans** (variable `wght` 100–900) |
| Display font: Newsreader / Libre Bodoni | Too subdued for hero numbers | **Fraunces** (variable `opsz`/`SOFT`/`WONK`/`wght`) |
| Pattern: "Scroll-Triggered Storytelling" / "Portfolio Grid" | Long-scroll landing logic — wrong shape for a TikTok-style feed | **Mobile Editorial Feed** (Section 1 of MASTER) |
| Style: "Flat Design" | Kills the editorial weight | **Editorial Maximalism** (Section 2) |
| CTA color: hot pink `#EC4899` | Reads "AI product" + on the banned pink-purple list | **Catalst Coral `#FF4D4D`** |
| Modal overlay: `backdrop-filter: blur(4px)` | Glassmorphism creep on content surfaces | Solid dim `rgba(11,11,15,0.40)` (see Section 7) |
| Single 5-color palette | Spec calls for 7 base + 9 mood-lens tints | See Section 3 of MASTER for the full token set |
| Palette path: `design-system/catalst-market/MASTER.md` | `CLAUDE.md` specifies `design-system/MASTER.md` | Move the file up; delete the sub-dir |

When you regenerate, diff the new raw output against the current `MASTER.md` and **only port over things the skill genuinely improved on** (e.g., a better-justified shadow scale). Keep the constraint-driven overrides intact.

Hard constraints to re-verify on every regen, copied from `CLAUDE.md`:

- No Inter or Roboto as primary font
- No glassmorphism on content surfaces (Story Cards, BuildSheet body, modal interiors). Liquid Glass tokens (`--glass-chrome`, `--glass-floating`, `--glass-toast`) are **only** for floating chrome — see Section 5 of MASTER.
- No dark mode as default
- No cyberpunk / HUD / terminal / Bloomberg aesthetics
- No AI-purple or pink-purple gradients
- No dense data-table layouts
- No globe / 3D background

## 4. Per-page overrides

Per-route deviations from MASTER live in `design-system/pages/<page>.md` and **override** `MASTER.md` at that page's scope. The lookup logic is documented at the top of `MASTER.md`:

> When building a specific page, first check `design-system/pages/<page>.md`. If that file exists, its rules **override** this Master file. If not, strictly follow MASTER.

Suggested overrides as the build progresses (from Section 13 of MASTER):

- `pages/feed.md` — the home `/` mood feed
- `pages/story.md` — `/story/[id]` deep link
- `pages/forge-bridge.md` — the BuildSheet → `/forge?seed=` handoff
- `pages/daily-brief.md` — the first-load modal

Create these only when a page first ships — don't pre-populate empty stubs. The `.gitkeep` in this directory exists solely to keep the empty folder under version control.

---

## 5. Commit discipline

When you regenerate MASTER:

1. Run the search command with `--persist`.
2. Manually move/merge from `design-system/catalst-market/MASTER.md` into `design-system/MASTER.md`.
3. Re-apply every hand-tuning override from the table above.
4. Commit as `refactor(design-system): regen MASTER with <reason>` and call out in the body what changed vs. the previous master and which overrides were re-applied.
5. Run `/design-review` against an existing page to verify nothing broke.

Never regenerate and commit in a single mechanical step. The skill is a starting point, not the source of truth.
