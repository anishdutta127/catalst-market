# CLAUDE.md — catalst-market

## Identity
Catalst Market. The Market layer of Catalst OS. A live, mood-first feed of what's happening in the business + startup world, designed for the general public. Every story is a copy-able opportunity. The wedge is the **Build with Catalst** CTA on every story — one click into Verdania ideation.

## Audience
Not VCs. Not founders only. The general public who want to be hustlers. India + global. Mobile-first. They scroll like TikTok, decide like Robinhood, copy like Pinterest.

## Atomic unit
A **Story** — not an "event."
- Founder face (or company logo)
- One-line *what they did*
- One number that pops (the headline metric, big serif type)
- 30-second *how* (3 micro-bullets max)
- *Why it matters now* (1 line)
- Copy-able angle (the build prompt)
- Build CTA → Verdania `/forge?seed=<story-id>`

## Three switches (the IA spine)
1. **Stage** — Empires (public + unicorns) | Builders (seed–B) | Bootstrappers (no VC)
2. **Industry** — bento of industry tiles, swipeable, mood-art per tile
3. **Mood** — "I'm feeling..." → 9 curated lenses:
   - 🔥 Blowing up • 🌱 Underdog wins • 🪙 Bootstrapped to millions
   - 🚀 Overnight rockets • 🧠 Quiet builders • 💡 Copy-able ideas
   - 🎯 Founders like me • 💸 Big money moves • 🇮🇳 India shipping

Switches compose. Default route: home → mood picker → stage toggle visible → industry filter optional.

## Visual law
NOT Bloomberg. NOT terminal. NOT purple gradients on white.
- Consumer-pop refined. Editorial photography energy. Mobile feed feel.
- Design system is generated by `uipro-cli`, persisted to `design-system/MASTER.md`. NEVER hand-pick colors/fonts. If MASTER.md does not match the page need, write a `design-system/pages/<page>.md` override.
- Anti-patterns to refuse: Inter as primary font, neon purple, glassmorphism on every card, AI-slop gradients, terminal aesthetics, dense data-table layouts.

## Habit loop (non-negotiable)
- Streak counter (visible always)
- Daily Brief modal on first load (3 missed signals from your tracked moods)
- Daily Quest (+50 XP) — surfaces in Insight Bar
- XP from: opening app, claiming a build, sharing a story, completing a quest
- Toast for new live signals matching your mood/industry
- Push (later) — only for Pulse-tracked entities, max 1/day

## Build CTA — the wedge
Every story card has it. Always visible. Always one tap.
- Tap → opens BuildSheet (modal)
- BuildSheet shows: codename, wedge hypothesis, target user, first build, house assignment
- Confirm → POST to Catalst OS forge endpoint, deeplink to `/forge?seed=<story-id>`
- BuildSheet copy is type-specific (funding ≠ launch ≠ M&A — see `lib/angles.ts`)

## Stack (locked)
- Next.js 15 (App Router)
- TypeScript strict
- Tailwind v4
- Framer Motion
- Zustand for client state
- React Query for server state
- Supabase Postgres for stories + user state (per gstack default)
- Composio MCP for ingestion (Crunchbase, ProductHunt, HN, GitHub, RSS)
- Vercel deploy
- Bun runtime in dev (matches Catalst OS)

## Working agreements
- Direct, no neutral-options. Recommend, don't survey.
- One commit per concern. Conventional commits. `feat:` `fix:` `chore:` `docs:`
- gstack `/review` ≥ 9 AND `/design-review` ≥ 9 before any merge.
- TDD on every `lib/` module. UI is exempt but Storybook-tested.
- Use `using-git-worktrees` for parallel batch builds.
- If you generate >100 lines without writing a test, stop and write one.

## Don't
- Don't build a 3D-rotating data-globe as the centerpiece (the original Bloomberg-terminal pattern). The dotted-continent SVG hero on / IS approved — see design-system/pages/feed.md for spec. Globe is banned everywhere except /.
- Don't write content copy in Claude voice. Story copy is third-person, editorial, present-tense.
- Don't use the word "leverage." Don't use emojis as icons (use Lucide).
- Don't ship without a real Story shape — the type lives in `lib/types/story.ts` and changes propagate.
- Don't add subscription gating before MVP. Mana economy comes after we have 1k DAU.
- Don't compete on data depth (Crunchbase wins). Compete on emotional clarity + the build CTA.

## Definition of done (per batch)
1. Tests pass (`bun test`)
2. `/review` ≥ 9
3. `/design-review` ≥ 9 against the relevant `design-system/pages/<page>.md`
4. Visual check at mobile (375px), tablet (768px), desktop (1440px)
5. Lighthouse ≥ 90 on perf + a11y
6. One PR per batch — commit messages tell the story
7. Demo recorded (Loom-style, 60 sec) before merge to main

## Compass for ambiguity
When stuck, ask: "Does this make a 22-year-old in Mumbai who has never met a VC feel like they could build something tomorrow?" If no → cut it.
